突然觉得，自习室是个很难得的地方，有一群人想提高自己的人在这里学习，而且环境条件也不错。大一大二时只觉得“我在寝室照样学的下去”“走去图书馆要花十几分钟呢”结果消耗了太多时间在无意义的事情里（比如寝室里无尽的“哈哈哈”）。
今天室友又来问我关于电脑的问题，.caj文件怎么打开等等。我曾很热情的帮他们解决问题，后来发现不管自己在干什么，别人一句“过来帮我看一下！”我就得打断自己正在做的事。有几次太烦了不想理他们，却被冷言嘲讽。后来我遇到这种情况一律回答“自己上百度查”，他们就说“我不知道怎么查啊”。有时也会心软帮他们看看，比如这次的“.caj文件怎么打开”。但是他的电脑卡的二B，点了右键等了好几分钟没反应，按下CTRL＋SHIFT＋ESC也没反应，按下CTRL＋ALT＋DEL还是没反应。（windows电脑需要不断的维护清理，如果没有好的使用习惯就会越用越卡，对很多人的解决方案就是卡的不行了就重装系统）重启电脑又等了好几分钟，问他把查看.caj的应用程序装在哪儿了，他说：“不知道，你再装一遍不行啊？”好有道理，无言以对。又装了一遍，提示某个插件版本太低无法正常安装，告诉他要下插件，他说：“我之前怎么就装上了。”我只有回答不知道，目前按正常步骤就是这样的结果。他折腾了一会儿说：“喔，我记得当时我好像随便改了一下然后就装上了。”我快疯了，你怎么能随便改呢？！你都不知道自己在改什么，改完也不知道自己在装什么，装完也不知道在哪儿，我给你解决方案你又不信，我TM书没看完题没做完论文没写完刚晾了衣服就屁颠屁颠跑你这儿，我文明用语。

英语


我发现，当我在读英语时，某句话中遇到一个生词，这时我查词典，知道了这个词的释义，然后我就继续往下读了。


回想在学习汉语时遇到这种情况——某个字／词不认识时，通过工具书查到它的释义、发音甚至例句、渊源，最后还会想想什么时候会用到这个字／词或者什么时候用过这个字／词。下意识地这样来了个全套，而且自己并没有什么异样的感觉，仿佛这一切都是自然而然的。


为什么到了英语这儿，查了释义之后就停止了呢？仅仅满足于大致知道这块儿讲的什么，然后就匆匆忙忙的往下继续了呢？


那么，我该做什么？

关于翻译的思考


当我学习 Python 时，我选择看英文原版，中文译版作为参考用。为什么？第一，当我看原版时，我会更容易弄清作者想表达的东西和逻辑。从一种语言翻译到另一种语言肯定会损失很多有趣的东西。第二，Python 的语句都是用英文写的。当我写一个语句时，大脑中很容易有当初看书时的印象。第三，英文术语更加浅显易懂。有一次和同学讨论某电源芯片的用法时，对方说“斩波”、“自举”，我表示不懂，画出来一看我说这就是技术手册上的“buck”、“boost”嘛。有一段时间我十分抵触那些“翻译过来的专业术语”，我觉得这些乱糟糟的东西把简单事情复杂化了。比如“整型”、“长整型”、“双精度型”、“浮点型”，第一次接触这几个概念我当场懵逼。后来看原版书知道“integer types”、“floating types”，只觉得中国人何苦为难中国人……


最近我的观点有所改变，统一的专业术语是有必要的，这样专业人士之间在沟通时会方便一点，不用在几种语言之间切换，对双方的外语水平要求也有所降低。但这样是好的吗？我不知道。之前写Python 笔记时遇到专业术语一律写英文，例如 parameter 、function、method 等等，现在我会把有确定翻译的术语写成中文，只在第一次写出时后加括号注明英文，例如“参数（parameter）”。然而有些词我并没有找到比较好的翻译，比如 method 。方法？好奇怪……


我很喜欢翻译，喜欢用一种语言把另一种语言信达雅的再现出来。当我跃跃欲试想把看到的YouTube视频翻译下来时，才感到自己能力不足。“Ever wonder if romantic relationships are getting increasingly more difficult to navigate there days?”就这一句折腾半天都没撸通顺，而且 navigate 也不知道该如何翻译。
